## 0. 이 문서의 위치

`70-brand-identity`(톤) + `71-design-system`(토큰)을 **회화적 언어**로 구체화한다. "어떤 일러스트 스타일인가", "사주 원국은 어떻게 그리나", "명반(자미두수)은 어떻게 표시하나"를 결정한다.

**원칙.**
- **추상·기하 > 구상·캐릭터.** 사람 얼굴·점괘 이모지 없음.
- **데이터 시각화는 장식이 아니라 도구.** 모든 선·면에 정보가 실린다.
- **한 페이지 한 스타일.** 일러스트 + 3D + 사진 혼용 금지.

## 1. 일러스트·이미지 원칙

### 1.1 스타일

**택1: 기하학적 추상 라인 아트.**
- 선 기반 1.5px 두께. `color-ink-900` 단색.
- 면은 최소. 면이 들어가면 `color-neutral-100` 또는 `color-accent-900`의 10% opacity만.
- 원·사각·점·선 조합. 곡선은 절제.
- 참조: Arc browser 마케팅, Figma Config 키비주얼, Linear 블로그 일러스트.

**금지 스타일:**
- ❌ 3D 클레이 렌더 (blob 스타일)
- ❌ 이소메트릭 도시 일러스트
- ❌ 플랫 캐릭터 + 표정
- ❌ 픽사·디즈니풍 캐릭터
- ❌ 오컬트 삽화 (눈동자·피라미드·수정구)

### 1.2 일러스트 용도 계층

| 계층 | 용도 | 복잡도 |
|---|---|---|
| L1 Hero | 랜딩·마케팅·인쇄 | 복잡도 높음, 기하 구성 |
| L2 Section | 각 기능 섹션 제목 위 | 중간, 3~5 요소 |
| L3 Spot | 빈 상태·에러·도움말 | 단순, 1~2 요소 |
| L4 Icon | 메뉴·버튼 | 최소, Lucide 기반 |

### 1.3 그리드·정렬

- 모든 일러스트는 8px 그리드 위에 구축.
- 핵심 요소는 중심 또는 3분할 지점.
- 가능한 한 **수직·수평** 우선, 사선은 15° 단위로만.

## 2. 도메인 차트 · 9종 스펙

운명 프로덕트의 정체성을 가장 강하게 드러내는 부분. 9종 핵심 시각화의 규격.

### 2.1 C1 — 사주 원국 표 (Saju WonGuk Table)

**용도:** 명리 엔진의 메인 시각.
**구조:** 4열(연·월·일·시) × N행(천간·지지·지장간·...) 그리드.

```
         연주        월주        일주        시주
천간      甲          丙          戊          壬
(갑)      (갑)        (병)        (무)        (임)
지지      子          寅          戌          子
지장간    壬·癸       戊·丙·甲    辛·丁·戊    壬·癸
```

**토큰:**
- 각 셀 크기: 모바일 72×88, 데스크톱 96×112
- 폰트: 한자 display serif(`font-serif` heading-2), 한글 루비 body-sm
- 일주 열은 1.5px 테두리로 강조 (일간이 본인의 축)
- 오행에 따라 상단 4px 라인(`color-element-*`)
- 지장간은 아래쪽에 작게(space 분할 구분선)

**상호작용:**
- 각 글자 클릭 → popover로 풀이·오행·십신 설명
- trace 펼치기 → 계산 근거

### 2.2 C2 — 오행 파이 차트 (Ohaeng Pie)

**용도:** 오행 분포를 직관적으로.
**스타일:** 도넛형. 내부 ring 두께 24%.

- 5조각. 각 오행 컬러 `color-element-*`
- 레이블: 외부 라벨 `오행명 · 비율%` 가로 라인
- 중심: 가장 강한 오행 이름 큰 글자 + "가장 우세한 오행"
- Variance: 편중도 높으면 원 둘레에 경고 점.
- 작은 비율(< 5%)은 "기타"로 묶거나 확대 보기

### 2.3 C3 — 십신 레이더 (Sipsin Radar)

**용도:** 십신(비견·겁재·식신·상관·편재·정재·편관·정관·편인·정인) 10축 레이더.
**스타일:** 십각형, `color-accent-900` 0.15 fill + 1px stroke.

### 2.4 C4 — 자미두수 명반 (Ziwei Palace Grid)

**용도:** 12궁 배치. 자미두수의 핵심 시각.
**구조:** 4×4 그리드의 외곽 12칸(가운데 2×2 비움 or 명주 정보).

```
┌───┬───┬───┬───┐
│命 │父 │福 │田 │
├───┼───┴───┼───┤
│兄 │        │官 │
├───┤  CORE  ├───┤
│夫 │ (명주) │奴 │
├───┼───┬───┼───┤
│子 │財 │疾 │遷 │
└───┴───┴───┴───┘
```

**토큰:**
- 각 궁: 가로 1fr, 세로 160px 이상
- 궁 이름은 상단 serif heading-3
- 주성(14주성): 중단, mono 숫자 + 한자
- 보조성: 작은 칩 다중 배치
- 사화(四化): 모서리 삼각 마크
- 명궁(命宮)은 1.5px border + `color-accent-900` 미세 틴트

### 2.5 C5 — Goldschneider 48주 휠 (48-Week Wheel)

**용도:** 48주 중 본인 주차 + 인접 주차 표시. 원형 타임라인.
**구조:** 원 12분할(별자리) × 각 분할 4분할(주차) = 48 섹터.

- 본인 섹터 `color-accent-900` fill
- 인접 섹터 accent-200 fill (앞뒤 1주씩, 큐스프 표시)
- 외곽에 별자리 심볼(12), 내곽에 주차 숫자
- 중심: 본인 주차 키워드 (예: "Week 3 — The Dreamer")

### 2.6 C6 — Enneagram 9각형 (Enneagram Diagram)

**용도:** 9개 유형 + 화살표 (통합/분열 방향).
**구조:** 정9각형, 연결선은 특정 규칙(1→7→5→8→2→4→1, 3→6→9→3).

- 본인 유형 노드 크게, `color-accent-900`
- 윙: 인접 노드 중간 크기
- 화살표: 통합 방향은 실선, 분열 방향은 점선
- 각 노드 hover 시 유형 설명 popover

### 2.7 C7 — MBTI 4축 바 (MBTI Axis Bars)

**용도:** 4개 축의 치우침과 강도.
**스타일:** 수평 바 4개. 중앙 0%, 좌우 양 끝 100%.

```
E ├──────────●────────────────┤ I    (내향 61%)
S ├────────────────●──────────┤ N    (직관 58%)
T ├──────●────────────────────┤ F    (사고 72%)
J ├──────────────────●────────┤ P    (인식 57%)
```

- 바는 18px height, `color-neutral-100` 트랙.
- 채워진 부분 `color-accent-900`.
- 위치 닷(●)은 선택된 방향의 비율 위치.
- 각 축 라벨은 양 끝 글자 + 우측에 "내향 61%" 라벨.

### 2.8 C8 — Trace Viewer

**용도:** 엔진의 `trace` JSON을 계층형으로.
**스타일:** 들여쓰기 + 접기 화살표 + mono 폰트.

- 각 노드 `label: value`, 값이 시간일 땐 하이라이트
- 경고(warnings)는 `color-signal-warning` 아이콘 + 메시지
- 클릭 시 해당 계산 단계의 도식 팝업

### 2.9 C9 — 교차 키워드 맵 (Cross Keyword Cloud)

**용도:** 5엔진에서 동시에 추출된 키워드.
**스타일:** 태그 클라우드가 아닌 **그리드 칩 배치**.

- 5엔진 공통(3+개 엔진에서 등장) 키워드: 큰 칩, fill `color-accent-900`/10%, 좌측에 엔진 마크 칩
- 2엔진 공통: 중간 칩, outlined
- 1엔진 전용: 작은 칩, 연한 텍스트
- 배치는 중요도 좌상→우하

### 2.10 차트 공통 규약

- **반응형.** 모든 차트는 모바일에서 스크롤·핀치 없이 파악 가능해야 함.
- **복사 가능.** 차트 하단에 "이미지 복사" 버튼.
- **접근성.** 모든 차트에 대응 텍스트 버전 제공 (screen reader용).
- **상호작용 표준화.** hover/tap은 popover, 길게 누름은 상세.

## 3. 커스텀 아이콘

### 3.1 도메인 아이콘 리스트 (자체 제작)

| 아이콘 | 용도 | 설계 힌트 |
|---|---|---|
| 10천간 | 원국 표·칩 | 한자 기반 + 오행 색 점 |
| 12지지 | 원국 표·칩 | 동물 아이콘 대신 한자 + 방위점 |
| 24절기 | 타임라인 | 간단한 씨앗/잎 기호 변주 |
| 14주성 (자미두수) | 명반 | 별(★) + 위첨자 |
| 12궁 (자미두수) | 명반 라벨 | 한자 + 기능 심볼 |
| 9 유형 (애니어그램) | 뱃지 | 숫자 + 모티브 | 
| 16 유형 (MBTI) | 뱃지 | 4자 + 색 |

모두 라인 아트 1.5px, 24×24 기본. currentColor.

### 3.2 시스템 아이콘

Lucide 사용. 선별 기준:
- 모든 아이콘은 **1.5px stroke**
- 채움 금지. outline only.
- 의미 중복 피함 (예: user, person, profile 중 `user`만 선택).

## 4. 카드·리포트 장식

### 4.1 장식 규칙

- **섹션 구분자:** 얇은 구분선 + 작은 심볼 (예: 명리 섹션엔 천간 '甲' 미니, 애니어그램엔 9 뱃지).
- **인용·포에틱 블록:** 세리프 큰 글자 + 좌측 `color-accent-900` 3px 바
- **강조 구절:** `highlight` 컴포넌트. 배경 `color-accent-900` 10%, inline padding.
- **테이블:** 굵은 보더 금지. 1px 수평선만.

### 4.2 섹션 제목 패턴

```
─────── 명 리 ───────
당신의 사주 원국
```

- 상단 작은 라벨(uppercase, tracking widest, `text-xs`, `color-text-tertiary`)
- 큰 제목(serif heading-1)
- 선택적 리드 문장(body-lg, `color-text-secondary`)

## 6. 애니메이션·전환

### 6.1 기본 원칙

- **단순 > 화려.**
- 1초 내 완료. 사용자 대기 금지.
- prefers-reduced-motion → 모두 off.

### 6.2 주요 전환

| 상황 | 모션 | 토큰 |
|---|---|---|
| 페이지 전환 | 80ms fade + 2px 상향 | duration-fast, ease-out |
| 카드 호버 | shadow 강화 + 2px 상향 | duration-fast |
| 팝오버 열림 | scale 0.96→1 + fade | duration-normal, ease-spring |
| 토스트 등장 | 슬라이드 20px↑ | duration-normal |
| 리포트 섹션 스크롤 | 섹션 제목 sticky, 하단 섹션 fade-in | — |
| 차트 entrance | 0.3초 stroke draw | duration-slow, ease-out |
| 설문 다음 문항 | 12px 슬라이드 | duration-fast |

### 6.3 절대 쓰지 않을 모션

- bounce/elastic 큰 진폭 (유치함)
- 회전 360도 로딩
- 다중 동시 애니메이션 (시각 소음)
