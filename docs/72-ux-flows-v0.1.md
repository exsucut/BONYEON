## 0. 이 문서의 위치

사용자가 앱 안에서 거치는 **경로(flow)** 와 화면 간 **관계(IA)** 를 규정한다. 화면 내부 구성·컴포넌트는 `71-design-system`, 시각 스타일은 `73-visual-language`에서.

**원칙.**
- 모든 핵심 행위는 **3탭 이내** 도달.
- **정보 → 결제** 순서. 가치 체감 전에 결제 벽을 세우지 않는다.
- **중단·재개 가능.** 설문, 프로필 입력은 언제든 저장 후 나갔다 돌아와도 이어진다.
- **다크 패턴 금지.** 해지·환불·연락 경로 은닉 금지.

## 3. F1 — 첫 방문부터 첫 리포트

### 3.1 단계

```
1. [Landing /]         "다섯 관점으로 당신을 봅니다" + [시작하기]
        │
        ▼
2. [Onboard intro]     회원가입 유도 vs 체험 옵션
        │              "지금 체험해 보기 (회원가입 없이 1회)"
        ▼
3. [Mini input]        생년월일/시간/출생지 — 필수만
        │              "모르면 넘겨요" (시간 모름 체크박스)
        ▼
4. [계산 중...]        2~4초, 계산 trace 티저 슬라이드
        │
        ▼
5. [요약 리포트]       5개 섹션 카드. 각 카드는 1~2문단만 공개.
        │              상세는 회원가입 후 잠금 해제.
        ▼
6. [전환 포인트]       "이 리포트를 저장하고 상세 보기"
                       → 소셜 로그인 1탭
```

### 3.2 체험 → 가입 전환 설계

- 체험 결과는 **익명 세션**에 저장(쿠키). 가입 시 자동 연결.
- 가입하지 않으면 **24시간 후 세션 폐기**, 재방문 시 다시 입력 유도.
- 가입 전환 CTA는 **요약 리포트의 마지막에만 등장**. 본문 중간 삽입 금지.

### 3.3 KPI

- 랜딩 → 체험 시작 클릭률 (목표 15%+)
- 체험 시작 → 리포트 도달률 (목표 85%+)
- 리포트 도달 → 회원가입 전환율 (목표 20%+)

## 4. F2 — 프로필 생성 (생년월일시)

### 4.1 입력 필드 (순서대로 한 화면씩)

1. **이름(별명)** — 단일 인풋. 본인/상대/가족 구분용. 실명 요구 안 함.
2. **생년월일** — 년/월/일. 한국 달력 기본.
3. **양력/음력 스위치** — 음력 선택 시 윤달 체크박스.
4. **출생 시각** — 시/분. "시간을 모릅니다" 토글. 토글 ON이면 분은 숨김·시주 없음.
5. **성별** — 남/여/기타. 명리·자미두수 대운 방향에 영향.
6. **출생지** — 도시 프리셋(서울·부산·대구 등 주요 20개) + "직접 입력" (위도/경도). 프리셋 기본값: "서울".
7. **고급 옵션(접힘)** — 자시 관례 3종, 진태양시 사용 여부. 기본값: unified + 진태양시 ON.

### 4.2 진행 UX

- 1필드 1화면(모바일) / 한 페이지 스크롤(데스크톱) 하이브리드. 모바일은 단계별, 데스크톱은 한 화면.
- 상단 진행바. 뒤로가기 상시 가능.
- 각 단계에 **"왜 이게 필요한가"** 인라인 도움말 (작게).
  예: "출생지? 진태양시 보정을 위해 경도가 필요합니다."

### 4.3 검증

- 생년월일 범위 1900–2100 (만세력 스펙 §2).
- 음력 선택 시 윤달 2번 이상 불가능한 해는 체크박스 disabled.
- 출생 시각 없음 선택 시 워닝: "시주 없이 계산되며 일부 해석이 제한됩니다."

### 4.4 완료 후

- 자동으로 5개 엔진 병렬 계산. 2~5초.
- 완료되면 `/app/profiles/:id`로 이동, `#summary` 섹션에 초점.

## 5. F3 — 설문 (MBTI·Enneagram)

### 5.1 공통 UX

- 1문항 1화면(모바일) / 10문항 1페이지(데스크톱). 사용자 선택 시에 따라.
- 진행바, 저장하고 나가기, 건너뛰기(최대 10%까지).
- **예상 소요시간 고지**: MBTI 자체문항 약 12분, Enneagram 약 15분.

### 5.2 MBTI 설문

- 자체 개발 60문항 (각 축 15문항). 7점 리커트.
- 각 축 점수 합산 → 16유형 판정 + 각 축 강도(%).

### 5.3 Enneagram 설문

- 자체 개발 135문항 (각 유형 15문항). 5점 리커트.
- 9 유형 점수 → 1위 유형, 2위 윙 후보.

### 5.4 세션

- 10문항 제출마다 자동 저장.
- 중단 시 "이어서 하기" 배너 상단 상시 노출.

### 5.5 결과

- 결과 화면 진입 시: 유형 요약 + "왜 이 유형인지" trace 보기(답변 분포 차트).

## 6. F4 — 리포트 뷰어

### 6.1 레이아웃

```
┌──────────────────────────────────────────────────────┐
│ [프로필 이름]   생성 2026-04-20            [공유][…]│
├────────────┬─────────────────────────────────────────┤
│ Summary ●  │  # 요약                                 │
│ 명리       │                                         │
│ 자미두수   │  교차 키워드 · 한눈에 보는 프로필         │
│ Goldsch.   │                                         │
│ MBTI       │  [엔진 5카드 간략]                        │
│ Enneagram  │                                         │
│ ─────      │                                         │
│ 교차 해석  │  # 명리                                 │
│            │  ...                                    │
└────────────┴─────────────────────────────────────────┘
```

### 6.2 섹션 로딩

- Summary는 즉시. 엔진별 섹션은 순차 로드(스켈레톤 → 내용).
- 계산이 끝나지 않은 섹션은 "계산 중..."으로 표시하되 스크롤 가능.

### 6.3 각 엔진 섹션 표준 구조

1. **요약 1줄** (불릿 금지, 완전한 문장)
2. **핵심 시각화** (도메인 차트 — 사주 원국, 명반, 48주 휠 등)
3. **본문 해석** (3~7 문단, 2~5개 서브섹션)
4. **trace 펼치기** (접힌 상태, 클릭 시 계산 내역)
5. **결제 벽** (무료 사용자에게 본문 2~3문단 뒤 부드럽게)

### 6.4 교차 해석 섹션 (#cross)

- 5엔진 공통 키워드 추출 → 태그 클라우드 (단, 클리셰 단어 필터링)
- "명리와 애니어그램이 공통으로 말하는 것" 형식의 소섹션 3~5개
- 모순 지점 — "명리는 외향을, MBTI는 내향을 시사합니다. 당신은 상황에 따라 다른 얼굴을 가집니다."

### 6.5 상호작용

- 각 해석 문단 하단에 👍/👎 + "왜 이렇게 생각하세요?" (선택)
- 용어 클릭 시 popover로 풀이
- 섹션 단위 저장(북마크)

## 7. F5 — 결제 (구독·단건)

### 7.1 진입점

- 리포트 본문 중 결제 벽
- `/pricing`
- 설정 → 결제
- 기능 시도 시 모달 (예: "PDF 다운로드는 구독/단건")

### 7.2 결제 화면 구조

```
┌─────────────────────────────────────────────┐
│ 요금제 비교 (무료 / 단건 / 월 / 연)           │
├─────────────────────────────────────────────┤
│ 선택된 요금제 요약                            │
│   - 가격, 주기, 포함 기능                     │
│   - 다음 결제일                               │
├─────────────────────────────────────────────┤
│ 결제 수단 선택                                │
│   ● 카드   ○ 간편결제   ○ 계좌이체           │
├─────────────────────────────────────────────┤
│ 약관 동의 (이용약관·개인정보·결제)             │
├─────────────────────────────────────────────┤
│ [결제하기 · 최종 금액 9,900원]                 │
└─────────────────────────────────────────────┘
```

### 7.3 요금제 기본안 (상세는 `50-payments`)

- **무료**: 요약 리포트 + 광고
- **단건 리포트**: 3,900원 (1건, 30일 열람)
- **월 구독**: 9,900원/월
- **연 구독**: 99,000원/년 (2개월 할인)

### 7.4 결제 이후

- 즉시 엔터타이틀먼트 갱신 → 잠겨있던 섹션 부드럽게 펼침
- 영수증 이메일 발송
- 해지 경로 안내 (어느 메뉴에서 해지 가능한지)

## 8. F6 — 궁합 분석

### 8.1 진입

- "궁합" 메뉴 → 두 프로필 선택 (저장된 것 or 즉석 입력)

### 8.2 결과 섹션

1. 전체 궁합 점수 (하나의 숫자 대신 다차원 — 에너지·의사소통·생활리듬·장기궁합)
2. 명리 궁합 (오행 보완, 일주 관계)
3. 자미두수 궁합 (명궁·부처궁 비교)
4. 별자리 궁합 (Goldschneider)
5. MBTI 궁합 (공식 궁합표 회피 — 자체 해석)
6. Enneagram 궁합

### 8.3 톤

- 점수를 단정짓지 않음. "잘 맞는다" 대신 "보완적입니다" 또는 "마찰 지점은 X"

## 9. F7 — 공유

### 9.1 공유 대상

1. **요약 카드 이미지** (1080×1350, 인스타 스토리/피드)

## 11. F9 — 해지·탈퇴·환불

### 11.1 해지

- 설정 → 결제 → "구독 해지"
- **2탭 이내** 도달. 다크 패턴 금지.
- 확인 단계 1개만. "해지 이유" 설문은 **선택**, 스킵 가능.
- 해지 후 이용 종료일까지 기능 유지. 중간 환불은 법정 케이스만.

### 11.2 탈퇴

- 설정 → 계정 → "탈퇴"
- 비밀번호(또는 소셜 재인증) + 이유(선택) + "모든 데이터가 30일 후 영구 삭제됨" 고지
- 탈퇴 직후 소프트 삭제, 30일 이내 복구 가능. 이후 하드 삭제.
- 결제 기록은 법정 보존.

### 11.3 환불

- 전자상거래법 준수: 디지털 콘텐츠도 7일 이내 미사용 시 환불 가능.
- "사용 개시" 시점 = 리포트 잠금 해제(유료 섹션 공개). 그 전까지는 환불 가능.
- 환불 요청 페이지 제공. 고객센터 이메일 2영업일 이내 응답.

## 12. 빈 상태·오류·로딩

### 12.1 빈 상태 패턴

| 상황 | 화면 |
|---|---|
| 프로필 0개 | "첫 프로필을 만들어 보세요" + [프로필 만들기] |
| 설문 미진행 | "MBTI 설문을 통해 교차 해석이 완성됩니다" + [시작하기] |
| 궁합 프로필 부족 | "최소 2개의 프로필이 필요합니다" |

### 12.2 오류 패턴

| 유형 | 메시지 | 액션 |
|---|---|---|
| 네트워크 오류 | "연결이 끊어졌습니다." | [다시 시도] |
| 계산 실패 (엔진) | "계산 중 문제가 발생했습니다. 개발팀에 자동 전송됨." | [다시 시도] |
| 입력 불가능 날짜 | "1900년 이전은 지원하지 않습니다." | 인풋 상단 helper |
| 모호한 DST | "해당 시각은 표준시/서머타임이 모호합니다. 선택해 주세요." | 선택지 2개 |
| 결제 실패 | "결제가 취소되었습니다. 카드를 확인해 주세요." | [다시 결제] |

### 12.3 로딩 패턴

- 스켈레톤: 카드·테이블 골격만 회색 placeholder
- 스피너: **크리티컬한 저장/결제만**. 일반 데이터 페치에는 스켈레톤.
- 엔진 계산 중: 계산 중인 엔진 이름 + trace 티저 ("입춘 시각 찾는 중...")
