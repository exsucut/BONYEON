**Version:** 0.1 (initial)
**Status:** Draft
**Standard:** WCAG 2.1 Level AA
**Audience:** 본인(개발자), 프런트 기여자, QA
**Last updated:** 2026-04-20

---

## 0. 이 문서의 위치

접근성은 "나중에 붙이는 기능"이 아니라 **기본 품질**이다. 본 문서는 WCAG 2.1 AA 준수를 목표로, 운명 플랫폼의 구체적 구현 규칙을 정의한다. 디자인 토큰(`71-design-system`)과 차트(`73-visual-language`)는 본 문서의 요건을 충족하도록 설계되어야 한다.

**원칙.**
- **명시적 합격 기준.** "접근성 고려함"은 의미 없음. WCAG 성공 기준 번호로 추적.
- **자동 검사 + 수동 검수.** axe-core CI + 월 1회 수동 스크린리더 워크스루.
- **한국어 특수 고려.** 한자·한글 혼용, 스크린리더의 한자 읽기 이슈 대응.

## 1. 준수 레벨과 범위

### 1.1 목표

- **WCAG 2.1 AA** (AAA는 선택적으로 리포트 본문에서).
- **KWCAG 2.2** (한국형 웹 접근성) 동시 충족.

### 1.2 범위

| 화면 | 우선순위 |
|---|---|
| 랜딩, 가격, 법적 페이지 | **P0** |
| 회원가입·로그인 | **P0** |
| 프로필 생성 폼 | **P0** |
| 리포트 뷰어 본문 | **P0** |
| 결제 플로우 | **P0** |
| 설문 UI | **P1** |
| 차트 시각화 | **P1** (대응 텍스트 필수) |
| 관리자·설정 | **P2** |

## 2. 색 대비

### 2.1 대비 비율 요건 (WCAG 1.4.3)

| 대상 | 요건 | 우리 목표 |
|---|---|---|
| 일반 텍스트 | 4.5:1 | ≥ 4.5:1 |
| 큰 텍스트 (≥ 18px bold, ≥ 24px regular) | 3:1 | ≥ 4.5:1 |
| UI 요소 테두리, 아이콘 | 3:1 | ≥ 3:1 |
| 포커스 인디케이터 | 3:1 | ≥ 3:1 |

### 2.2 토큰 쌍 대비 검증

디자인 시스템의 주요 대비 쌍 검증 (`71-design-system §2` 기반).

| 전경 | 배경 | 대비 | 판정 |
|---|---|---|---|
| `color-ink-900` `#141414` | `color-neutral-25` `#FAFAF7` | 18.3:1 | AAA ✅ |
| `color-neutral-600` `#6B665C` | `color-neutral-25` `#FAFAF7` | 5.0:1 | AA ✅ |
| `color-neutral-400` `#9A9488` | `color-neutral-25` `#FAFAF7` | 2.7:1 | ❌ 작은 텍스트 금지 (캡션·디저블드만) |
| `color-neutral-0` `#FFFFFF` | `color-accent-900` `#1E3A8A` | 10.8:1 | AAA ✅ |
| `color-accent-900` `#1E3A8A` | `color-neutral-25` `#FAFAF7` | 10.2:1 | AAA ✅ |
| `color-signal-error` `#8C3A2E` | `color-neutral-25` `#FAFAF7` | 6.4:1 | AA ✅ |

> **규약:** `color-neutral-400`은 **오직 placeholder·disabled·caption 장식**에만 사용. 본문 텍스트로 쓰면 자동 axe 검사에서 탈락.

### 2.3 오행 컬러 대비

오행 5색은 채도 낮춘 버전(`§73-visual-language §2.2` 참조)이라 흰 배경에서 텍스트로 쓰기엔 대비가 낮다. **텍스트에 오행 색 직접 사용 금지**, 장식(라인·칩 배경)에만 사용.

### 2.4 색에만 의존하지 않기 (WCAG 1.4.1)

- 그래프의 구분은 **색 + 패턴 or 라벨**.
- 오류 메시지는 **색 + 아이콘 + 텍스트**.
- 링크는 **색 + 밑줄** (마우스오버 아닌 항상 표시 원칙, 단 리포트 본문 긴 글은 예외로 호버 시 밑줄).

## 3. 키보드 네비게이션

### 3.1 기본 요건 (WCAG 2.1.1, 2.4.3, 2.4.7)

- **모든 인터랙티브 요소** tab 순서로 접근 가능.
- **포커스 순서**는 시각적 순서와 일치.
- **포커스 인디케이터**는 명시적으로 표시 (`focus-visible` CSS로 키보드 전용).
- **포커스 트랩**: 모달·드로어 열릴 때 내부로 제한, 닫을 때 원 요소로 복귀.

### 3.2 표준 단축키

| 키 | 동작 |
|---|---|
| `Tab` / `Shift+Tab` | 다음/이전 포커스 |
| `Enter` | 버튼·링크 실행 |
| `Space` | 체크박스·버튼 |
| `Esc` | 모달·드로어·팝오버 닫기 |
| `↑ ↓ ← →` | 리스트·라디오·탭 이동 |
| `Home/End` | 리스트 처음/끝 |
| `⌘K / Ctrl+K` | 커맨드 팔레트 (프로필 전환) |
| `?` | 단축키 안내 모달 |

### 3.3 Skip Links

페이지 최상단 첫 Tab에 "본문으로 건너뛰기" 링크 노출.

### 3.4 차트 키보드

각 차트(9종)는 Tab으로 진입 가능해야 하며, Enter 시 **텍스트 데이터 버전** 모달 또는 대체 텍스트 표시.

## 4. 스크린리더

### 4.1 지원 대상

- macOS VoiceOver
- Windows NVDA
- iOS VoiceOver
- Android TalkBack

### 4.2 의미 있는 마크업 (WCAG 1.3.1)

- **랜드마크**: `header`, `nav`, `main`, `footer`, `aside` 명시.
- **헤딩 계층**: 페이지당 `h1` 1개, 순서 어기지 않음.
- **리스트**: `ul`/`ol`. `div` 나열 금지.
- **버튼 vs 링크**: 행위(클릭으로 어떤 동작)는 `button`, 이동은 `a`.
- **ARIA**는 시맨틱 HTML을 대체할 수 없을 때만. 기본은 HTML.

### 4.3 라벨 규칙 (WCAG 2.4.6, 3.3.2)

- 모든 폼 요소에 `<label>` 연결. 아이콘 버튼은 `aria-label`.
- placeholder는 라벨이 아니다. 반드시 label 동반.
- 이미지에 `alt`. 장식 이미지는 `alt=""`.

### 4.4 동적 변경 (WCAG 4.1.3)

- 계산 완료·결제 성공 등은 `aria-live="polite"` 영역에서 안내.
- 오류 메시지는 `aria-live="assertive"` + `role="alert"`.

### 4.5 한자 읽기 문제

한국어 스크린리더는 한자를 **한국어 음독**으로 읽지 못하거나 일본 음으로 읽는 경우가 있다. 리포트의 한자 전용 표기에는 `ruby` 또는 `aria-label`로 한글 병기:

```html
<span aria-label="경금">庚金</span>
```

또는 더 명확히:

```html
<ruby>庚金<rt>경금</rt></ruby>
```

## 5. 폰트·타이포그래피

### 5.1 기본 크기

- 본문 최소 **16px** (브라우저 기본과 동일).
- `rem` 기반 스케일로 사용자 브라우저 기본 크기 설정 존중.

### 5.2 확대 지원 (WCAG 1.4.4, 1.4.10)

- 최대 **200%** 확대해도 레이아웃·기능 유지.
- **reflow** (1.4.10): 320px 폭에서도 수평 스크롤 없이 읽기 가능.

### 5.3 줄높이·자간

- 본문 줄높이 최소 **1.5** (WCAG 1.4.12).
- 문단 간격 본문 크기의 **2배** 이상.
- 자간 **0.12em** 이상 설정 가능 (사용자 CSS).

### 5.4 텍스트 간격 오버라이드 테스트

사용자 스타일시트로 자간·줄높이·문단 간격을 늘렸을 때 텍스트 잘림·겹침 없어야 함.

## 6. 폼 접근성

### 6.1 라벨·설명·오류 (WCAG 3.3.1, 3.3.3)

모든 인풋은 3요소 세트:

```html
<div>
  <label for="birthdate">생년월일</label>
  <input id="birthdate" aria-describedby="birthdate-hint birthdate-error">
  <p id="birthdate-hint">양력/음력은 다음 단계에서 선택합니다.</p>
  <p id="birthdate-error" role="alert">1900년 이후로 입력해 주세요.</p>
</div>
```

### 6.2 필수 입력 표시

- 필수: 라벨에 "(필수)" 텍스트 + `aria-required="true"`. 빨간 별표만 금지.
- 선택: "(선택)" 텍스트.

### 6.3 오류 복구 (WCAG 3.3.4)

- 결제·탈퇴·설문 제출 등 중요 제출 전 **검토 단계** 제공.
- 결제 오류 시 입력값 유지, 문제 필드에 포커스.

### 6.4 자동 완성 (WCAG 1.3.5)

- 이메일·이름 필드에 `autocomplete` 적절히 지정 (예: `autocomplete="name"`, `autocomplete="email"`).

### 6.5 생년월일시 피커 특수

- 네이티브 `input[type="date"]`는 스크린리더 호환 가변. **커스텀 피커에도 fallback으로 직접 입력** 허용.
- 음력 체크박스는 날짜 인풋과 명시적 연결 (`aria-describedby`).

## 7. 모션·애니메이션

### 7.1 prefers-reduced-motion (WCAG 2.3.3)

CSS:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 1ms !important;
    transition-duration: 1ms !important;
    scroll-behavior: auto !important;
  }
}
```

JS(Framer Motion): `useReducedMotion` 훅으로 모든 모션 bypass.

### 7.2 자동 재생 금지

- 자동 재생되는 영상·오디오 없음 (v1 영상 없음).
- 자동 슬라이드 캐러셀 금지. 수동 제어만.

### 7.3 깜빡임 (WCAG 2.3.1)

- 초당 3회 이상 플래시 금지. 우리는 그런 UI 없지만 규약으로 명시.

## 8. 국제화·언어 처리

### 8.1 lang 속성 (WCAG 3.1.1, 3.1.2)

```html
<html lang="ko">
  <p>본문 내 영어 <span lang="en">example</span> 표기.</p>
  <p>한자 독음 필요 시 <ruby>庚金<rt lang="ko">경금</rt></ruby>.</p>
</html>
```

### 8.2 숫자·날짜·통화

- 날짜 포맷: 한국 로케일 (`2026년 4월 20일`).
- 통화: `₩9,900` 또는 `9,900원`. 스크린리더용 `aria-label="9900원"`.

### 8.3 RTL 대비

v1은 한국어만이지만, 아랍어·히브리어 로컬라이제이션 대비해 **컴포넌트는 `dir` 속성 의식적으로** 작성.

## 9. 이미지·차트·아이콘

### 9.1 alt 텍스트 (WCAG 1.1.1)

- 의미 있는 이미지: 내용 요약.
- 장식 이미지: `alt=""` 또는 `role="presentation"`.
- 복잡한 시각(차트·인포그래픽): 짧은 alt + 장문 설명은 인접 요소 또는 `aria-describedby`.

### 9.2 차트 대체 텍스트

각 차트(9종)는 **텍스트 버전을 함께 렌더**. 시각 장애인은 텍스트로, 시각 사용자에게는 `sr-only` 클래스로 숨김.

예: 오행 파이 차트의 대체 텍스트
```
오행 분포: 화 35%, 목 28%, 토 20%, 금 10%, 수 7%.
가장 우세한 오행은 화(불)이며, 수(물) 비율이 가장 낮습니다.
```

### 9.3 아이콘-only 버튼

`aria-label` 필수. 시각 텍스트 없는 버튼은 항상 접근 가능 이름 제공.

```html
<button aria-label="공유">
  <Icon name="share" />
</button>
```

## 10. 라이브·동적 콘텐츠

### 10.1 로딩 상태

- 스켈레톤: `aria-busy="true"` + 완료 시 `aria-live` 영역에 "로드 완료".
- 장기 작업(결제·계산): 진행률을 live region에 반영.

### 10.2 토스트·알림

- 중요 알림: `role="alert"` (auto-announce).
- 비침투 알림: `role="status"` / `aria-live="polite"`.

### 10.3 모달

- `role="dialog"` + `aria-modal="true"`.
- `aria-labelledby`로 모달 제목 참조.
- 포커스 트랩, Esc 닫기.

## 11. 자동 검사 파이프라인

### 11.1 CI 단계

```
PR → Playwright e2e + axe-core 결과 첨부 → 새 violation 0이어야 머지
```

- **도구**: `@axe-core/playwright` 또는 `axe-playwright`.
- **기준**: Critical·Serious 위반 0건. Moderate·Minor는 경고.

### 11.2 Lighthouse CI

- 각 주요 페이지에서 Lighthouse Accessibility 점수 **90+ 유지**.
- PR에서 10 이상 하락 시 차단.

### 11.3 단위 레벨

- 컴포넌트 Storybook + axe addon (도입 시).
